函数式编程与 JS 异步编程、手写 Promise

一、函数式编程

概念：同面向对象并列，是一种编程思维、范式、风格。需要抽象运算过程。
注意：并不会提高程序性能，因为其中大量使用闭包

//简单的函数式
function add(num1,num2){
    return num1+num2;
}
三个概念：

1.函数是一等公民：他也是一个普通的对象

  • 函数可以存储在变量中
  • 函数作为参数
  • 函数是作为返回值

2.高阶函数（抽象问题）

  • 可以把函数作为参数传递给另一个函数
  • 可以把函数作为另一个函数的返回结果（此处可能产生闭包）
  • 常用的高阶函数：数组的一系列方法：forEach、map、filter、every、some、find|findIndex、reduce、sort、...
//抽象一个只执行一次的函数，可用在支付时
function once(fn){
    let done = false
    return function(){
        if(!done){
            done = true
            return fn.apply(this,arguments)
        }
    }
}

3.闭包:函数和其周围的状态引用捆绑在一起形成闭包

  • 可以在另一个作用域中调用一个函数的内部函数并访问到该函数的作用域中的成员
//例子
function makeFn(){
    let msg = 'loved'
    return function(){//返回的函数中引用了内部成员则形成闭包
        console.log(msg)
    }
}
const fn = makeFn()
fn()

总结：因为内部函数被外部引用，导致内部作用域成员无法释放，所以形成闭包

二、纯函数：相同的输入一定得到相同的输出，没有任何可观察的副作用

  • 纯函数：slice 返回数组中的指定部分，不会改变原数组
  • 不纯函数：splice 对数组进行操作返回该数组，会改变原数组
优点、缺点：
  • 可缓存：相同输入=>相同输出。因此可存储第一次的结果
  • 可测试：
  • 并行处理：不需要访问共享的内存数据,并行环境下可任意运行纯函数
  • 副作用：一些可变的参数配置导致纯函数不纯
参照lodash中 _memoize 模拟记忆函数
//例子
function memoize (f) {
    let cache = {}
    return function(){
        let key = JSON.stringfy(arguments)
        cache[key] = cache[key]||f.apply(f,arguments)
        return cache[key]
    }
}

三、柯里化：

  • 当一个函数有多个参数的时候先传递一部分参数调用它（这部分参数以后永远不变）
  • 然后返回一个新的函数接收剩余的参数，返回结果
lodash中的柯里化函数功能
//例子
const _ = require('lodash') 
// 要柯里化的函数 
function getSum (a, b, c) {  
    return a + b + c 
} 
// 柯里化后的函数 
let curried = _.curry(getSum) 
// 测试 
curried(1, 2, 3) 
curried(1)(2)(3) 
curried(1, 2)(3)

以上例子得出结论：我们可以提前传入一些常用参数做配置得到一个函数。最后传入剩余参数得到结果

模拟 _.curry 的实现:原理，返回一个存储部分参数的函数，直到所有参数真正传递完毕才调用
//例子
//es5
function curry(func){
    return curryFn(...args){
        if(args.length<func.length){
            return function(){
                return curryFn(...args.concat(Array.from(arguments)))
            }
        }
        return func(...args)
    }
}

//es6
const curry = (func) => {
    return curryFn = (...args){
        if(args.length<func.length){
            return (...args2)=>{
                return curryFn(...args.concat(args2))
            }
        }
    }
}

总结：
柯里化可以让我们给一个函数传递较少的参数得到一个已经记住了某些固定参数的新函数 这是一种对函数参数的'缓存'
让函数变的更灵活，让函数的粒度更小
可以把多元函数转换成一元函数，可以组合使用函数产生强大的功能

四、函数组合：

what？

  • 纯函数和柯里化很容易写出洋葱代码 h(g(f(x)))。如：获取数组的最后一个元素再转换成大写字母， .toUpper(.first(_.reverse(array)))，函数组合可以避免这种写法
  • 函数组合可以让我们把细粒度的函数重新组合生成一个新的函数
  • 函数组合就像用管接头链接一根根管道，每根管道就是一个细粒度的函数，接收参数和向下传递返回值，最后输出想要的结果

why？

  • 避免洋葱函数的层层嵌套。
  • 结合函数柯里化更直观的处理参数，同时可以结合细粒度的函数生成功能强大的函数
  • 可以方便的进行调试
  • lodash 中也有fp模块（封装了柯里化函数），且遵循函数优先，数据之后。以便于更好的使用函数组合

do？

  • 要点：
    1.被组合的函数需使用一个参数
    2.使用柯里化函数对多参数函数提前传入部分参数，以达到单参数目的
    3.柯里化结合函数组合实现管道调用，流水作业
    4.组合函数从右往左依次调用，接受一个参数，处理后传递向下一个函数
    5.满足结合律
    6.实现测试方法，准确定位管接处错误
  • 模仿fp.flowRight()实现函数组合

Point Free

Point Free：我们可以把数据处理的过程定义成与数据无关的合成运算，不需要用到代表数据的那个参 数，只要把简单的运算步骤合成到一起，在使用这种模式之前我们需要定义一些辅助的基本运算函数。

五、函子：

作用：在函数编程中控制副作用（异常处理，异常操作）
  • 容器：包含值和值的变形关系（这个变形关系就是函数）
  • 函子：一个特殊容器，通过普通对象实现，对象具有map方法，map方法可以运行一个函数对值进行处理
  • 基本结构如下：
class Container{
    conctructor(value){
        this._value =value
    }
    map(fn){
        return new Container(fn(this._value))
    }
}
1.完整的Functor函子：用函数式编程思想，不用new方法
class Container{
    static of(value){
        return new Container(value)
    }
    constructor(value){
        this._value = value
    }
    map(fn){
        return Container.of(fn(this._value))
    }
}

总结：

  • 函数式编程运算不直接操作值，由函子完成
  • 函子就是实现了一个map契约的对象
  • 函子就是一个放了值的盒子
  • 盒子暴露的接口即map方法传递处理值的函数（此函数为纯函数）
  • map方法返回一个包含新值的盒子（函子），满足链式调用

class MayBe{
    static of(value){
        return new Container(value)

    constructor(value){
        this._value = value

    map(fn){
        return this.isNothing() ? MayBe.of(null) : MayBe.of(fn(this._value))

    isNothing(){
        return this._value === null || this._value === undefined
    }
}

七、手写Promise：

基本要点：

1.Promise有三种状态，PENDING(等待)、FULLFILED(成功)、REJECTED(失败)，状态一旦改变则不可更改
2.Promise接受一个函数，这个函数(excutor)会在Promise内部立即执行,并且接受两个改变状态的参数resolve,reject,同时进行值的存储以便传递
3.Promise具有一个then方法，该方法接受成功和失败的回调函数：
（1）then方法的返回值仍然是promise以便与后续链式调用，
（2）then方法中会处理回调函数的返回值以便与传递给下一个then方法
（3）promise中会对then方法中回调函数的返回值进行验证，需要把任何值都最终处理成普通值通过promise向下传递
（4）如果then方法回调函数中返回值是当前promise实例，则抛出错误
4.如果是延时任务，调用then方法时状态还未改变，那么Promise内部会存储then方法的回调函数，以便于状态改变时调用
5.promise可以多次调用then方法，哪怕其中有异步任务，通过数组存储所有then中传入的回调函数；
6.需要对excutor和then做错误处理
7.其他要点有关promise的其他实例方法和静态方法，均在完整代码中，不做赘述

const REJECTED = 'REJECTED'
class MyPromise {

            this.reject(e)
        }

    reason = undefined;

    failCallback = []
    resolve = value => {//5.改变状态
        if(value instanceof MyPromise){//30.如果resolve值是promise,那么递归调用以得到普通值
            value.then(this.resolve,this.reject)
        }
        if (this.status !== PENDING) return //6.状态一旦更改便无法再次改变
        this.value = value  //8.获取状态改变时传入的参数存储以便于向后传递
        this.status = FULLFILLED
        // this.successCallback&&this.successCallback(this.value) //12.等到状态改变时再执行回调
        while (this.successCallback.length) {  //14.依次执行多次调用then方法的函数
            // this.successCallback.unshift()(this.value)  
            this.successCallback.unshift()()  //23.存值是已经传递了参数，并且我们调用的是外层函数，因此不用再传值
        }
    }
    reject = reason => {
        if (this.status !== PENDING) return
        this.reason = reason
        this.status = REJECTED

        while (this.failCallback.length) {
            this.failCallback.unshift()()
        }

                        // successCallback(this.value) 
                        let x = successCallback(this.value) //16.存储回调参数的返回值以便向后传递.注：如何向后传递，考虑用then返回的promise对象去把返回值向下传递。考虑把整块代码移入promise对象中，既能同步执行代码，又可以拿到resolve方法以便于传递返回值。
                        // resolve(x) 
                        //resolvePromise(x,resolve,reject)  //17.由于x值有可能为普通值  也有可能为promise对象，因此做判断后再传递
                        resolvePromise(promise2, x, resolve, reject)  //18.promise不允许then方法返回自身实例，因此该传入promise2做判断
                        //19.此处无法获取promise2实例，巧妙利用setTimeout解决问题
                    } catch (e) {
                        reject(e)
                    }
                })
            } else if (this.status === REJECTED) {
                setTimeout(() => {
                    try {
                        let x = failCallback(this.reason)
                        resolvePromise(promise2, x, resolve, reject)
                    } catch (e) {
                        reject(e)
                    }
                })
            } else {//10.处理异步情况,调用then方法时装态还未更改，则存储回调函数，去resolve中状态更改时触发

                // this.failCallback = failCallback

                            let x = successCallback(this.value)
                            resolvePromise(promise2, x, resolve, reject)
                        } catch (e) {
                            reject(e)
                        }
                    })
                })

                    setTimeout(() => {

                        } catch (e) {

                        }
                    })

            }
        })

                if (index === array.length) {
                    resolve(result)
                }
            }
            for (let i = 0; i < array.length; i++) {
                let current = args[i]
                if (current instanceof MyPromise) {
                    current.then(value => addResult(i, value),reason=>reject(reason))
                }else{
                    addResult(i, current)
                }
            }
        })
    }
    static resolve (value){//28.如果是promise直接返回，如果是普通值则转换为promise返回
        // if(value instanceof MyPromise)return value  //此处不做判断，通过对then方法做递归求值 直到返回值不为promise
        return new MyPromise((resolve)=>resolve(value))
    }
    finally(callback){//31. [1].无论成功或失败，callback都可执行 [2].把当前的value或reason向下传递  [3].返回一个promise对象  [4].callback返回值可以是promise，并且会等待这个promise结束后再向后执行
        //32.考虑如何获取当前promise的状态，调用this.then()方法
        return this.then(value=>{
            // callback()   //此时 如果callback是一个promise,无法等待.
            // return value
            return Promise.resolve(callback()).then(()=>value)
        },err=>{
            // callback()
            // throw err 
            return Promise.resolve(callback()).then(()=>{throw err})
        })

    }
    catch(failCallback){//29.借用then方法实现catch
        return this.then(undefined,failCallback)
    }
}
