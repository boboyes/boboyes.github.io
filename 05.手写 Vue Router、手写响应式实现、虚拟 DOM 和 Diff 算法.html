三. Vue响应式原理

  • 数据驱动

    1. 数据响应式
      • 数据模型仅仅是普通的javascript对象，而当我们修改数据时，视图会进行更新，避免了繁琐的dom操作，提高开发效率
    2. 双向绑定
      • 数据改变，视图改变；视图改变，数据也改变
      • 我们可以使用v-model在表单元素上创建双向数据绑定
    3. 数据驱动是Vue最独特的特性之一
      • 开发过程仅仅需要关注数据本身，不需要关心数据是如何渲染到视图

  • 基于Object.defineProperty()实现数据劫持

        //单个属性
    //dom
    <div id='haha'></div>

    //js
    let data = {
        msg = ''
    }
    let vm = {} //模拟vue实例

    Object.defineProperty(vm,'msg',{
        set(newValue){ //存值，劫持存值函数，以更新视图

                return

            //值变化更新dom
            document.querySelector('#haha').textContent = data.msg
        },
        get(){  //取值
            console.log('get:',data.msg)
            return data.msg
        }
    })

    //多个属性
    let data = {
        msg:'aa',
        haha:'bobo'
    }
    function proxyData(data){
        Object.keys(data).forEach(key=>{
            Object.defineProperty(vm,key,{
                set(newValue){ //存值，劫持存值函数，以更新视图
                    if(newValue === data[key]){
                        return 
                    }
                    data[key] = newValue
                    //值变化更新dom
                    document.querySelector('#haha').textContent = data[key]
                },
                get(){  //取值
                    console.log('get:',data[key])
                    return data[key]
                }
            })
        })
    }

  • 发布订阅模式

        //注册事件，触发事件
    class EventEmitter{
        constructor(){
            this.subs = Object.create(null) //创建空对象以存储订阅事件，使用此方式增加性能
            //{'eventType':[event1,event2]}

            //注册事件【订阅者】
            $on(eventType,handler){
                this.subs[eventType] = this.subs[eventType] || []  //若已注册则赋值，未注册赋值空数组，以便添加事件函数
                this.subs[eventType].push(handler)
            }

            //触发事件
            $emit(eventType){
                if(this.subs[eventType]){
                    this.subs[eventType].forEach(handler=>{
                        handler()
                    })
                }
            }
        }
    }

  • Observer 响应式处理

    1. 负责把data中的属性转换成响应式数据
    2. data中的某个属性也是对象，把该属性转换成响应式数据
    3. 数据变化发送通知
      结构类图：
        class Observer {
        constructor(data){
            this.walk(data)
        }
        walk(data){//1.遍历数据  （抽象遍历方法是为了方便递归调用，对象嵌套的值也需要监听）
            if(!data||typeof data!=='object'){
                return
            }
            Object.keys(data).forEach(key => {
                this.defineReactive(data,key,data[key])
            });

        }
        defineReactive(data,key,value){//2.将属性变为可监听的
            let that = this
            //劫持前先实例化dep,准备收集依赖
            let dep = new Dep()
            this.walk(value) //3.value是对象继续调用遍历方法添加setter/getter
            Object.defineProperty(data,key,{
                enumerable:true,
                configurable:true,
                get(){
                    //一旦数据被模板访问，则收集依赖，以便数据更新时视图更新
                    Dep.target && dep.addSub(Dep.target)  //注意在water实例化后才会触发add
                    return value  //若使用data[key]，会递归调用getter,报错
                },
                set(newValue){
                    if(newValue === value){
                        return
                    }
                    value = newValue
                    that.walk(newValue) //如果值被修改为对象也需要setter/getter
                    dep.notify()
                }
            })
        }
    }

    2. 负责页面的首次渲染
    3. 当数据结构变化后重新渲染视图
      结构类图：
        class Compiler{
        constructor(vm){
            this.vm = vm
            this.el = vm.$el
            this.compile(this.el)
        }
        compile(el){
            let childNodes = el.childNodes //类数组
            Array.from(childNodes).forEach(node=>{
                if(this.isTextNode(node)){
                    this.compileText(node)
                }else if(this.isElementNode){
                    this.compileElement(node)
                }
                if(node.childNodes&&node.childNodes.length){//如果节点下还有子节点
                    this.compile(node)
                }
            })
        }
        compileText(node){  //文本节点替换差值表达式
            // console.dir(node) //将节点已对象的方式打印
            let textContent = node.textContent
            let reg = /\{\{(.+?)\}\}/  //匹配花括号
            if(reg.test(textContent)){
                let key = RegExp.$1.trim()
                // console.log(key,'KEY')
                node.textContent = textContent.replace(reg,this.vm[key])

                //一旦该数据被编译到模板上，触发监听【把要做的事情注册到dep中，当值改变后就会触发注册的函数】
                new Watcher(this.vm,key,(newValue)=>{
                    node.textContent = newValue
                })
            } 
        }
        compileElement(node){  //找出元素节点中的属性节点，得出v-指令
            // console.log(node.attributes)
            Array.from(node.attributes).forEach(attr=>{
                console.dir(attr.name,'attr')
                let attrName = attr.name
                if(this.isDrective(attrName)){
                    attrName = attrName.substr(2)
                    let key = attr.value  //指令绑定的变量名
                    console.log(attr.value,'namee')
                    this.update(node,key,attrName)
                }
            })
        }
        update(node,key,attrName){
            let updateFn = this[attrName+'Updater'] 
            updateFn && updateFn.call(this,node,this.vm[key],key)
        }
        textUpdater(node,value,key){
            node.textContent = value
            new Watcher(this.vm,key,(newValue)=>{
                node.textContent = newValue
            })
        }
        modelUpdater(node,value,key){
            node.value = value
            new Watcher(this.vm,key,(newValue)=>{
                node.value = newValue
            })
            // 双向绑定
            node.addEventListener('input', () => {
                this.vm[key] = node.value
            })
        }
        isDrective(attrName){
            return attrName.startsWith('v-')
        }
        //判断节点类型
        isTextNode(node){
            return node.nodeType === 3  //文本节点类型
        }
        isElementNode(node){
            return node.nodeType === 1 //元素节点
        }
    }

        class Dep{
        constructor(){
            this.subs = []
        }
        addSub(sub){
            if(sub&&sub.update){
                this.subs.push(sub)
            }
        }
        notify(){
            this.subs.forEach(sub=>{
                sub.update()
            })
        }
    }

  • Wather

    1. 当数据变化触发依赖，dep通知所有的Watcher实例更新视图
    2. 自身实例化的时候往Dep对象中添加自己
      结构类图
        class Watcher{
        constructor(vm,key,cb){
            this.vm = vm
            this.key = key
            this.cb = cb

            //以下表示wather监听开启时那么就得通知Dep收集依赖

            Dep.target = this  //给依赖收集器判断标志，表示可以注入该依赖

            this.oldValue = this.vm[key] //去读一遍数据，用以触发getter，执行里面的dep依赖收集

            Dep.target = null  //已收集则置空

        }
        update(){ //订阅者的订阅内容提交   //此函数在dep notify时触发，即data属性setter是触发
            let newValue = this.vm[this.key]  //或许数据更新时的新值
            if(newValue===this.oldValue){
                return
            }
            this.cb(newValue)   //调用实例化wather时传入的回调函数，用以更新dom
        }
    }

总结：

  1. Vue 定义了一个vue构造函数，接受实例时的参数，将所有参数存储，将data转换成setter/getter并挂载到vue实例
  2. Observer 响应式处理，将$data中的所有数据转换成setter/getter, 并在此getter收集依赖，
    此setter中触发依赖
  3. Compiler 模板解析，解析html中的差值表达式和指令，替换成对应的data值，并且，一旦改值被替换到html中，那么开启watcher（只有渲染到页面中的数据才值得开启wather）
  4. Wather，wather开启后，将会通过重新get（即对应data属性赋值）的方式触发dep依赖收集
  5. dep,一旦数据发生改变，触发set里的dep.notify(),以触发wather注入的函数，以更新视图。

    1. 维护视图和状态的关系
    2. 复杂视图下提升渲染性能

    5. patch(oldVnode,newVnode)
        打补丁，把新节点中变化的内容渲染到真实的DOM,最后返回新节点作为下一次处理的旧节点
    对比新旧VNode是否是相同节点（比较key和sel[选择器]）
    如果不是相同节点，删除之前的内容，重新渲染
    如果是相同节点，判断新的VNode是否有text,如果有且和oldVnode的text不同，直接更新文本内容
    如果新的VNode有children,判断子节点是否有变化。判断子节点的过程使用diff算法
    diff过程只进行同层级的比较

    • 在pacth函数中，传入新旧Vnode,对比差异，把差异渲染到dom,并返回新的Vnode,作为下一次patch的oldVnode
    1. 首先执行模块中的钩子函数pre
    2. 如果新旧节点相同（即key和sel）,调用patchVnode()，查找节点差异并更新Dom
    3. 如果旧节点是DOM,则转换为Vnode,比较得出差异后使用createElm()把vnode转换为真实Dom，记录到vnode.elm中，然后将创建的DOM插入到parent中，移除老节点。
    4. 触发用户钩子create

  • patchVnode 函数：真正对比新旧节点的差异，将差异渲染到dom

    1. 执行用户的prepatch钩子函数
    2. 执行模块的create钩子，执行用户的create钩子
    3. 如果 vnode.text 未定义：
      • 新旧节点都有子节点，调用updateChild()，使用diff算法对比子节点并更新
      • 如果旧节点没有子节点，将新节点的子节点批量添加
      • 如果新节点没有子节点，将老节点的子节点批量移除
      • 如果老节点.text有值则清空
    4. 如果设置了vnode.text并和老节点text不相等，移除老节点的自己点，设置Dom的textContent为
      vnode.text
    5. 执行用户钩子postpatch

  • updateChildren是diff算法的核心,对比新旧节点的子节点，更新dom

    1. 在进行同级别节点比较时，首先会对新老节点数组的开始和结尾节点设置标记索引，遍历过程中移动索引以实现对比
    2. 当oldStartVnode / newStartVnode (旧开始节点 / 新开始节点) 或 oldEndVnode / newEndVnode (旧结束节点 / 新结束节点) 比较时：如果 是 sameVnode (key 和 sel 相同) ，调用 patchVnode() 对比和更新节点，把旧开始和新开始索引往后移动 oldStartIdx++ / oldEndIdx++ 或 把旧结束和新结束索引往前移动 oldStartIdx-- / oldEndIdx--
    3. 当oldStartVnode / newEndVnode (旧开始节点 / 新结束节点) 相同 ，调用 patchVnode() 对比和更新节点 ，把 oldStartVnode 对应的 DOM 元素，移动到右边
    4. 当oldEndVnode / newStartVnode (旧结束节点 / 新开始节点) 相同 ，调用 patchVnode() 对比和更新节点 ，把 oldEndVnode 对应的 DOM 元素，移动到左边 ，更新索引
    5. 若非以上四种情况：
      • 遍历新节点，使用 newStartNode 的 key 在老节点数组中找相同节点
      • 如果没有找到，说明 newStartNode 是新节点，创建新节点对应的 DOM 元素，插入到 DOM 树中
      • 如果找到了，判断新老节点的 sel 选择器是否相同：果不相同，说明节点被修改了 重新创建对应的 DOM 元素，插入到 DOM 树中 ；如果相同，把 elmToMove 对应的 DOM 元素，移动到左边
    6. 当老节点的所有子节点先遍历完 (oldStartIdx > oldEndIdx) 或 新节点的所有子节点先遍历完 (newStartIdx > newEndIdx)，循环结束
    7. 如果老节点的数组先遍历完(oldStartIdx > oldEndIdx)，说明新节点有剩余，把剩余节点批量 插入到右边；如果新节点的数组先遍历完(newStartIdx > newEndIdx)，说明老节点有剩余，把剩余节点批 量删除
