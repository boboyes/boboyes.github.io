    完整版： 同时包含编译器和运行时版本
    编译器： 用来将模板字符串编译成为JavaScript渲染函数的代码，体积大，效率低；含3000多行代码
    运行时： 用来创建Vue实例、渲染并处理虚拟DOM等的代码、体积小、效率高。基本上就是除去编译器的代码
    UMD: UMD版本通用的模块版本，支持多种模块方式。vue.js默认文件就是运行时+编译器的UMD版本
    CommonJS:配合老的打包工具比如Browserify或webpack1
    ES Module: 从2.6开始Vue会提供两个ESModule构建文件，为现代打包工具提供的版本
        esm格式被设计为可以被静态分析（编译时处理），所以打包工具可以利用这一点来进行‘tree-shaking’并将用不到的代码排除出最终的包

    1. 首次渲染的过程
        Vue 在进行了一些静态成员，实例成员的挂载后：new Vue()
    1. 开始调用内部的_init方法
    2. 接受用户传入的options 并合并到实例初始化时生成的$options
    3. 设置vue实例的代理对象
    4. 给vue实例挂载一些成员如：生命周期，事件监听，state等
    5. 最后调用实例的$mount方法传入el渲染页面，核心是生成render函数
    6. 关于mount方法：重写的$mount方法会判断用户是否传入render,没有传入则使用传入的template转为render函数, 如果都没有则使用传入的el或者动态创建一个div,然后再转为render函数[compileToFunctions],把转换好的render函数存储到$options
        1. 如果用户传入render()，那么不需要处理，直接调用mount方法
        2. 如果未传入render，则需要重写mount方法先将template或el转换为render，再调用mount

        3. 未重写的mount方法，即只含运行时的vue的mount方法：
            1.获取用户传入的el，在mount方法内部调用mountComponent
                1. 判断$options下的render函数是否准备好【出错提示和创建空vnode赋值给$options下的render】
                2. 如果存在调用生命钩子：beforeMount
                3. 定义updateComponent方法：更新挂载组件（即挂载render）
                    1.updateComponent内部实则调用vm._update(vm._render()，hydrating)方法更新
                       vm._render() 此_render就是上面用户传入的render或则通过模板转换的render ,执行它将会生成虚拟dom
                       vm._update() 是真正更新页面的方法
                4. wather：updateComponent是在wather中被调用的
                    1. 将传入的updateComponent赋值给this.getter,即watcher实例的getter方法，首次渲染时直接调用watcher的get方法，在内部最总调用updateComponent，执行_update更新视图
                5. wather执行完后调用mounted钩子
    1. 响应式的分析
        1. 在内部的_init方法中，调用initState()初始化了props/methods/data等
    2. 用户传入data则将其挂载到vue实例并调用Observe(data)进行响应式处理，如果未传入则创建空对象挂载调用Observe(data)
    3. observe(data, true /* asRootData */)是响应式的入口，在内部创建了new Observer()并返回实例ob
    4. 在Observer中,创建依赖对象Dep,将传入的属性值value通过Object.defineProperty()设置setter\getter,  如果value是对象则递归调用defineReactive设置setter/getter，如果value是数组单独对数组做响应式处理（实质是改写一些数组方法，在方法调用后触发notify()触发更新）
    5. 关于defineReactive=>收集依赖-触发notify()
        数据设置响应式时实例化Dep,mount时将wather注入Dep.target，然后在update更新组件渲染中get模板数据触发依赖收集，数据set改变时触发更新通知notify

      2. 模板编译的入口和过程
          1. 在$mount之前，把用户传入的el或template通过compileToFunction转换为render函数,存储到$options
        const { render, staticRenderFns } = compileToFunctions(template, {...})
    2. compileToFunctions由createCompiler()函数生成
        const { compile, compileToFunctions } = createCompiler(baseOptions)
    3. createCompiler 由 createCompilerCreator()函数生成
        createCompiler接收和平台相关的选项参数在内部和用户传入合并
        const createCompiler = createCompilerCreator(fun())
    
    4. createCompilerCreator内部的baseCompile做了三件事情
        1. 将模板转换成ast抽象语法树，以树形的方式描述代码结构
            const ast = parse(template.trim(), options)
        2. 优化抽象语法树
            if (options.optimize !== false) {
                optimize(ast, options)
            }
        3. 把抽象语法树生成（字符串形式）的js代码
            const code = generate(ast, options)
        4. 返回createCompiler
    
    真实过程：
        1. 在compileToFunctions中，如果已经编译出结果则进行缓存并且返回。
        2. 没有编译出结果则调用compile方法进行编译生成js代码字符串
            1. 合并options
            2. 在compiled方法中调用模板编译的核心函数baseCompile()生成compiled对象
                1. 将模板转换成ast抽象语法树，以树形的方式描述代码结构
                    const ast = parse(template.trim(), options)
                    1. 解析options
                    2. 解析模板parseHTML(template,{...})生成root ->借鉴了一个开源库
                    3. 返回ast对象root
                2. 优化抽象语法树
                    if (options.optimize !== false) {
                        optimize(ast, options)
                    }
                    1. 标记静态节点【对应的dom子树永远不发生变化，纯文本内容】
                    2. 标记静态根节点[如果一个元素内只有文本节点，此时这个元素不是静态根节点，优化成本大于收益]
                3. 把抽象语法树生成（字符串形式）的js代码
                    const code = generate(ast, options)
                    1. 申请静态根节点存储空间做预配置
                    2. 将静态根节点作标记并转换为字符串js代码存储到已申请的空间，将非静态根节点转换为字符串js代码
            3. 记录编译错误信息
            4. 返回compiled对象即{ render, staticRenderFns }
        3. 将js代码字符串转换成js代码,缓存并返回
            1. 将节点和静态根节点代码字符串分别转为js存储
