2.简易的状态管理方案
```js
    定义一个对象，对象里面存储state状态和更改状态的方法
    export default {
        state:{
            prop:'',
            ...
        }
        setState(){
            this.state.prop=newProp
        }
    }

    所有需要共享状态的组件都导入这个对象，他们共享这个对象的所有属性和方法，以此实现简易的状态统一管理
```
3.Vuex

  1. what?

    • 专门为Vue.js设计的状态管理库
    • 采用集中式的方式存储需要共享的状态
    • Vuex的作用是进行状态管理，解决复杂组件通信，数据共享
    • Vuex集成到了devtools中，提供了time-travals时光旅行历史回滚功能

  2. 何时使用

    • 非必要的情况不要使用vuex
    • 大型的单页应用程序
      • 多个视图依赖同一状态
      • 来自不同视图的行为需要变更同一状态

  3. 核心概念：Store State Getter Mutation Action Module

  5. state:

        export default new Vuex.Store({
        state:{
            count:1,
            msg:'haha'
        },
    })

    <template>
        <div id='app'>
            //直接获取state
            {{$store.state.count}}
            {{$store.state.msg}}

            //语法糖加计算属性
            {{count}}
            {{msg}}

            //语法糖加计算属性加重命名
            {{num}}
            {{message}}
        </div>
    </template>

    <script>
        //使用语法糖和计算属性便捷获取state
        import {mapState} from 'vuex'
        export default {
            computed:{
                //count:state => state.count
                ...mapState(['count','msg']),
                //命名state避免和组件中变量冲突
                ...mapState({
                    num:'count',
                    message:'msg'
                }),
            }
        }
    </script>

  7. mutations:

  8. actions:

        export default new Vuex.Store({
        //...state,getters,mutations省略
        actions:{
            increateAsync(context,payload){//context中含state,getter,commit成员
                setTimeout(()=>{
                    context.commit('increate',payload)
                },2000)
            }
        }
    })

    <template>
        <div id='app'>
            <button @click="$store.dispatch('increateAsync',2)">Action</button>

            <button @click="increateAsync(3)">Action</button>

            <button @click="increAsync(3)">Action</button>
        </div>
    </template>

    <script>
        //使用语法糖和计算属性便捷获取actions
        import {mapActions} from 'vuex'
        export default {
            methods:{
                //increateAsync:actions => actions.increateAsync
                ...mapActions(['increateAsync']),
                //命名action避免和组件中变量冲突
                ...mapActions({
                    increAsync:'increateAsync'
                }),
            }
        }
    </script>

        const myPlugin = store => {
        //store初始化后调用
        store.subscribe((mutation, state) => { //订阅store中的mutation
            //每次 mutation 之后调用
            //mutation 的格式为 {type, payload} type中：命名空间/mutation名

        })

        modules:{products},

    })

  13. Vuex-模拟

        let _Vue = null
    class Store {
        constructor(options){
            const {
                state = {},
                getters = {},
                mutations = {},
                actions = {}
            } = options
            this.state = _Vue.Observable(state)
            this.getters = Object.create(null)
            Object.keys(getters).forEach(key=>{
                Object.defineProperty(this.gettters,key,{
                    get:()=>getters[key](state)
                })
            })
            this._mutations = mutations
            this._actions = actions
        }
        commit(type,payload){
            this._mutations[type](this.state,payload)
        }
        dispatch(type,payload){
            this._actions[type](this,payload) //简单模拟直接传递this，能够获取context即可
        }
    }

    function install(Vue){
        _Vue = Vue
        _Vue.mixin({
            beforeCreate(){
                if(this.$options.store){
                    _Vue.prototype.$store = this.$options.store
                }
            }
        }) 
    }
    
    export default {
        Store,
        install 
    }

二.服务端渲染基础

  • 优点：
    • 用户体验好
    • 开发效率高
    • 渲染性能好
    • 可维护性好
  • 缺点:
    • 首屏渲染时间长
    • 不利于SEO
1.同构应用
  1. 同构应用：
    • 通过服务端渲染首屏直出，解决SPA应用首屏渲染慢以及不利于SEO的问题
    • 通过客户端渲染接管页面内容交互得到更好的用户体验
    • 此种方式成为现代化的服务端渲染，也叫同构渲染
    • 称之为服务端渲染应用，同构应用
  2. 同构渲染实例：
    1. Vue、reactde官方解决方案，有助于理解原理，但需要搭建环境，操作极其繁琐
    2. React生态的Next.js
    3. Vue 生态的Nuxt.js
  3. 同构渲染的开发条件限制
    • 浏览器特定的代码只能在某些生命周期的钩子函数中使用;
    • 一些外部扩展库可能需要特殊处理才能在服务端渲染应用中运行;
    • 不能再服务端渲染期间操作dom
    • 某些代码需要区分运行环境
  4. 同构渲染构建和部署的限制
    • 构建：需要构建两个端，客户端和服务端
    • 部署：只能部署在Node.js Server(Vue react之类)
  5. 服务端负载
    • 在Node中渲染完整的应用程序，相比仅仅提供静态文件的服务器需要大量占用CPU资源
    • 如果应用在高流量环境中使用，需要准备相应的服务器负载
    • 需要更多的服务端渲染优化工作处理
  6. 使用建议
    • 首屏渲染速度是否真的重要?
    • 是否真的需求SEO?
2.Nuxt.js 基础
  • 官网：https://zh.nuxtjs.org/
  • GitHub 仓库：https://github.com/nuxt/nuxt.js
  1. 是什么?
    • Nuxt.js 是一个基于 Vue.js 的服务端渲染应用框架，它可以帮我们轻松的实现同构应用。
    • 通过对客户端/服务端基础架构的抽象组织，Nuxt.js 主要关注的是应用的 UI渲染。
    • Nuxt.js 预设了利用 Vue.js 开发服务端渲染的应用所需要的各种配置。
    • 还提供了一种命令叫： nuxt generate ，为基于 Vue.js 的应用提供生成对应的静态站 点的功能。
    • 信这个命令所提供的功能，是向开发集成各种微服务（Microservices）的 Web 应用迈开的新一 步。
    • Nuxt.js 为 客户端/服务端 这种典型的应用架构模式提供了许多有用的特性，例如异步数据 加载、中间件支持、布局支持等非常实用的功能。
  2. 特性
    • 基于 Vue.js Vue、Vue Router、Vuex、Vue SSR 自动代码分层 服务端渲染 强大的路由功能，支持异步数据 静态文件服务 ES2015+ 语法支持 打包和压缩 JS 和 CSS HTML 头部标签管理 本地开发支持热加载 集成 ESLint 支持各种样式预处理器： SASS、LESS、 Stylus 等等 支持 HTTP/2 推送
  3. 使用nuxtJS

    2. yarn add nuxt
    3. 配置启动命令："dev": "nuxt"
    4. 新建pages文件夹->新建index.vue
