1. React 介绍

  • 状态容器

2. 核心概念和工作流程

Store: 存储状态的容器
View: 视图，HTML
Actions: 对象，描述状态进行的操作的类型
Reducers: 函数，操作状态并返回新的状态

6. 中间件

  • 中间件允许扩展redux应用
  • action会优先被中间件处理，然后再传递给reducer
  // 中间件模板代码
  export default store => next => action => {}

  // 中间件使用方式
  import { createStore, applyMiddleware } from 'reduc'
  import logger from './middlewares/logger'
  createStore(reducer,applyMiddleware(  //注册中间件
    logger
  ))


  //中间件书写  
  export default store => next => action => {
    if (typeof action === 'function') {
      return action(dispatch)
    }
    next(action)
  }

9. Mobx

  • 简单 可扩展的状态管理库

  • Mobx负责管理应用状态

  • Mobx 5 版本运行再任何支持ES6 proxy的浏览器， 不支持ie11, nodejs 6

  • Mobx 4 可以运行在任何支持ES5的浏览器上

  • Mobx 4 和 5 的API是相同的

1. 使用

  • 启用装饰器语法支持(方式一)

    1. 弹射项目底层配置： npm run eject
    2. 下载装饰器语法babel插件 npm install @babel/plugin-proposal-decorators
    3. 在package.json中加入配置
    "babel": {
  "plugins": [
    {
      "@babel/plugin-proposal-decorators",
      {
        "legacy": true
      }
    }
  ]
}

  • 启用装饰器语法支持(方式二)

    1. npm install react-app-rewired customize-cra @babel/plugin-proposal-decorators
    2. 在项目根目录下创建 config-overrides.js并加入配置
      const { override, addDecoratorsLegacy } = require("customize-cra");
  module.exports = override(addDecoratorsLegacy())
    1. 命令替换
      "scripts": {
    "start": "react-app-rewired start",
    "build": "react-app-rewired build",
    "test": "react-app-rewired test",
  }

  • 解决vscode装饰器语法警告
    修改配置："javascript.implicitProjectConfig.experimentalDecorators": true
